仍旧是关于毕设

2021-03-08
いろいろ
约 1144 字 预计阅读 3 分钟

到实际开始做的时候才发现要做的东西多麻烦……既需要了解Spring的使用，也需要了解前端页面各种库和webpack的使用……

现在，至关重要的是去开一个头！使用所谓的“敏捷开发”的方式吧，先整出东西来，再不断迭代——

step1

第一步是完成公交车信息的展示，具体来说有以下需求——

后端

  • 公交车信息的数据库设计和domain，dao的实现
  • 返回某市区县的公交车之前一段时间的速度和位置的servlet和不断更新该信息的servlet的实现（建立长连接？）
  • 实时接受公交车所发送的信息的servlet的实现
  • 公交车模拟模块的实现

前端

  • 展示某市区县的所有公交车实时信息功能的实现

step2

然后是查询某日历史信息的实现，这也意味着需要先完成公交车及站点等数据的增删改功能。

后端

  • 完成管理员的domain和dao层（它的数据库设计是容易的）
  • 完成公交车及站点等数据的增删改功能
  • 返回某市区县公交车某日所有信息的servlet的实现

前端

  • 展示某市区县公交车某日所有信息功能的实现

step3

进一步增强前端数据可视化的功能，提供热力图等玩意，以追踪特定时间或特定位置超速情况。

后端

无

前端

  • 完善数据可视化功能
  • 完成生成CSV文件功能

显然第一步是最难的，应当继续细分。

首先是建表，考虑到查询之前数据的需要，表中数据必须存储一个时间戳，即使第一步并不提供管理的功能。

--- 对应公交车的表
CREATE TABLE bus(
    bus_id INT AUTO_INCREMENT PRIMARY KEY,
    update_time TIMESTAMP,
    bus_number VARCHAR(7), -- 车牌号，长度为7位，形如豫A99999
    bus_type VARCHAR(20) -- 车型
);

--- 对应各个区域的表 
CREATE TABLE place (
    place_id INT AUTO_INCREMENT,
    update_time TIMESTAMP,
    place_name VARCHAR(10),
    lat DECIMAL,
    lont DECIMAL,
    PRIMARY KEY (place_id, update_time)
);

--- 对应公交车线路的表
CREATE TABLE bus_line (
    line_id INT AUTO_INCREMENT,
    update_time TIMESTAMP,
    station_id_list TEXT, -- 途径的station，为{data:[...]}的json数据（或者xml？），这里为了保证数据的顺序化，进行这样的折中处理
    PRIMARY KEY (line_id, update_time)
);

--- 对应驾驶员的表
CREATE TABLE driver (
    driver_id INT AUTO_INCREMENT PRIMARY KEY,
    update_time TIMESTAMP,
    driver_name VARCHAR(10),
    driver_sex BIT,
    driver_birthday DATE
);

--- 对应公交车站点的表
CREATE TABLE station (
    station_id INT AUTO_INCREMENT,
    update_time TIMESTAMP,
    lat DECIMAL,
    lont DECIMAL,
    PRIMARY KEY (station_id, update_time)
);

--- 对应公交车发送信息（当前位置，车辆ID）的表
CREATE TABLE time_pos(
    bus_id INT,
    update_time TIMESTAMP,
    lat DECIMAL,
    lont DECIMAL,
    PRIMARY KEY (bus_id, update_time),
    FOREIGN KEY (bus_id) REFERENCES bus(bus_id)
);

--- 管理员的表
CREATE TABLE the_admin(
    admin_id VARCHAR(32) PRIMARY KEY, -- 用作登录
    admin_passwd VARCHAR(32), -- 可以不存储明文
    admin_name VARCHAR(32)
);

--------------------------------------------------------


--- 公交车线路所处于的位置（一般来说只为同一个位置，但跨区的线路有多个）
--- 与之前的想法不同（通过位置确定所需站点，再通过站点确定线路），选择直接通过位置确定线路。
CREATE TABLE line_place (
    line_id INT,
    update_time TIMESTAMP,
    place_id INT,
    PRIMARY KEY (line_id, update_time, place_id),
    FOREIGN KEY (place_id) REFERENCES place(place_id)
);

--- 公交车的驾驶员，一个驾驶员可能可以驾驶多个公交车，一个公交车可能可以由多个驾驶员驾驶
CREATE TABLE driver_bus (
    driver_id INT,
    update_time TIMESTAMP,
    bus_id INT,
    PRIMARY KEY (driver_id, update_time, bus_id),
    FOREIGN KEY (driver_id) REFERENCES driver(driver_id),
    FOREIGN KEY (bus_id) REFERENCES bus(bus_id)
);
