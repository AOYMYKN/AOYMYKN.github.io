对CSP的复盘

2020-12-13 约 1697 字 预计阅读 4 分钟

这一次CSP感觉难度比以往的高一些，我只拿了200分就跑路了，本应该能拿一点分的第三题被放弃着实可惜……随便写点什么。

第一二题都挺水的，但是题目却是费尽心思地往复杂的出，看题目半小时，coding 5分钟。第一题毫无压力，计算总和，大于零输出它，小于零就输出零。

第二题麻烦很多，但也好解决。它可以简化为——给定n对数字i，j，其中j为0,1，现在定义一个k，k可为任意一个i，求这样的k，使这n对数字中，i < k的数对中的0的数量加上i >= k的数对中1的数量最大（我不想，也不会写严格的数学描述）。

暴力解法是容易联想的，令k遍历所有的i，在每一次遍历中，遍历所有数对，每次保存最大时的k。这个解法是O(n^2)的。

根据我的几个同学的成绩，n^2解法只能得到70分（题目要求n<=50000，这显然不是平方时间复杂度的算法可以接受的）。我可能在实现上有一些问题，只获得了30分。于是我对算法进行了改进，使用滑动窗口的思想，统计对于每个i，其出现0和1的次数（使用两个哈希表实现，因为i的取值是非常大的，桶接受不了）。获取其keySet并排序。滚动记录对每一个i，小于i的0的数量和大于等于i的1的数量并记录最大值时的i。这样就把算法优化到nlogn时间复杂度（来自于排序，如果题目规定输入有序的话则是线性时间复杂度）。100分get

第三题是我想要重点提及的题，它模拟创建文件并设置文件大小，更改目录的孩子限额，后代限额，删除目录这几个操作，操作有失败的可能（比如重复的创建，或超出限额），如果失败，则不进行操作（这个坑死人了！）。

我认为，第三题我没有做出来的原因是对数据结构抽象的不彻底，试图在业务代码里直接操作数据结构，我使用二叉树来模拟文件树（显然是一个多叉树）——

class Node {
    public final String name;
    // 应当不要在业务代码里操作数据结构
    private int actualSize; // 对于目录应当是0
    private int childSize; // 孩子的总大小，对于文件应当是0
    private int descendantSize; // 子代的总大小，对于文件应当是0
    private int childCapacity; // 孩子配额
    private int descendantCapacity; // 子代配额
    private Map<String, Node> children; // 用map实现多叉树，用名称进行匹配符合要求
    private Node father; // 方便回朔
    private filetype type;
    // 对每个创建方法，应当递归更新该Node的father的childSize和descendantSize
    // 应该添加一个boolean addFileCheck(int size)方法，检查假如在该Node（目录）下插入一个文件会不会超出配额
    ...
}

以创建文件为例子，假设输入是C /hello/happy/world/123 1234，这代表创建文件并设置其大小为1234，首先把输入分隔成"hello",“happy”,“world”,“123”。首先检查会不会操作失败，检查分为两步——先检查是否重复创建，且最后一个path是目录，再检查如果设置，其上级的各目录的配额能否满足。验证完了？找到需要创建文件/目录的地址，比如，假如已经存在/hello/happy/目录，就在happy文件夹递归创建其后的目录/文件（即world文件夹，123文件）。该创建的方法应当由数据结构抽象。如果匹配到了最后，则是更新操作，这里可以先remove，再重新插入，也可以递归更新其father……怎么舒服怎么来吧！记住，在编写代码的时候，不要直接想着优化，**提前优化是万恶之源！提前优化是万恶之源！提前优化是万恶之源！**等因为超时拿不到满分再去优化吧！

第五题应该是一道线段树题，题目很容易看懂，可惜这个数据结构我还没学……总之写了个爆破的解法，应该能拿20分，但是我在出分前就跑路了，不知道拿到没有。

道路还长着呢，继续学习！不过这考试是不会再参加了，再您的见！
